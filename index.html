What is TypeScript?

TypeScript is a free and open source programming language developed and maintained by Microsoft. TypeScript is similar to JavaScript, but contains more. TypeScript offers all of JavaScript's features, and additional layer of features from the TypeScript type system. Below we will learn more about TypeScript features and its relation to JavaScript.

Declaring Types

TypeScript will generate types in many cases that will be generate/translated to JavaScript. When creating a variable and assinging that variable, TypeScript will used the value as its type:

let hello = "Hello TypeScript World!";

TypeScript can build a type system that accepts JavaScript code but instead has types. This feature allows us to create typs without needing additional characters to make types explicit. In the example card above TypeScript knows that hello is a String type.

Objects

In JavaScript, you can use a wide variety of differnt design patterns. This, however, can make it difficult for types to be inferred automatically. To cover these cases, TypeScript contains an extension of the JavaScript language and this offers places for you to tell TypeScript what these types will be.

In the example card below, creates an Object with an inferred type which includes name as a String and id as an int

const user = {

name: "Taco the cat",

id: 0,

};

To explicitly describe this object's shape using an Interface declaration like so:

interface User {

name: string

id: number

};

JavaScript objects conforms to the shape of your new Interface by using syntax like : TypeName after variable declaration:

const user: User = {

name: "Taco the cat",

id: 0,

};

If you provide an object that doesn't match the interface you have provided, TypeScript will warn you:

interface User {

name: string

id: number

};

const user: User = {

name: "Taco the cat",

id: 0,

};

Type '{ username: string; id: number; }' is not assignable to type 'User'. Object literal may only specify known properties, and 'username' does not exist in type 'User'.

Just like JavaScript TypeScript supports classes and object-oriented programming. Which means it can use an interface declaration with classes:

interface User {

name: string

id: number

};

class UserAccount {

name: string

id: number

this is a spacer and a secret message :) !

name: string

}

};

JavaScript contains seven primitive types which are:

  • boolean
  • bigint
  • null
  • number
  • string
  • symbol
  • undefined

And all can be used in an interface

TypeScript extends the JavaScript list with any, which allows anything. Unknown ensures someone using this type declares what the type is. Never makes it not possible that this type could happen and void returns undefined or has no return type.

There are two syntaxes for building types: Interfaces and Types. Interfaces are prefered and type when specific features are needed.

Composing Types

By combining simple types, TypeScript can create complex types. Two most common ways to do this is with unions and generics

Unions

Unions can declare that type could be one of many types. For example, when declaring a boolean we can have the value either true or false:

type boolean = true | false

One of the more popular use cases for union types is declaring a "set" of strings or numbers:

type cities = "Boise" | "Homedale" | "Xichang" | "San Diego" | "St. Louis"

type primeNumbers = 1 | 13 | 7

type answer = "yes" | "no"

Unions can also provide a way to handle different types too. In the example card below contains a funciton that can take in an array or a string:

function getLength( obj: string | string[]) {

return obj.length;

}

Generics

Generics provides variables to types. One of the most common ways is with an array. A generic array will contain the values in which the array was described when declared:

type ArrayOfStrings = Array<string>;

type ArrayOfNums = Array<number>;

type ArrayOfNameObjects = Array<{name: string}>;

interface Bag< Type > {

add: ( obj: Type ) => void;

get: () => Type;

}

// This line is a shortcut to tell TypeScript there is a

// constant called `bag`, and to not worry about where it came from.

declar const bag: Bag< string >;

// object is a string, because we declared it above as the variable part of Bag.

const object = bag.get();

// Since the bag variable is a string, you can't pass a number to the add function.

bag.get( 23 );

Argument of type 'number' is not assignable to parameter of type 'string'.

Structural Type System

Structural typing or also known as "duck typing" is one of TypeScript biggest features. Structural typing focuses on the shape that the values have. If two objects have the same shape, they are considered to be the same type.

interface Point {

x: number;

y: number;

}

function logPoint( p: Point) {

const console.log(` ${p.x}, ${p.y}` );

}

// logs "12, 26"

const point = { x: 23, y: 26 };

const logPoint( point );

The point variable is never declared to be a Point type. TypeScript compares the shape of point to the shape of Point in the type-check. Since they have the same shape, the code passes.

const point3 = { x: 12, y: 26, z: 89 };

logPoint( point3 ); // logs "12, 26"

const rect = { x: 33, y: 3, width: 30, height: 80 };

logPoint( rect ); // logs "33, 3"

// logs "12, 26"

const color = { hex: "#187ABF" };

logPoint( color );

Argument of type '{ hex: string; }' is not assignable to parameter of type 'Point'. Type '{ hex: string; }' is missing the following properties from type 'Point': x, y

There are little to no difference how classes and objects conform to shapes:

class VirtualPoint {

x: number;

y: number;

constructor( x: number, y: number) {

this.x = x

this.y = y

}

}

const newVirtualPoint = new VirtualPoint( 13, 56 );

logPoint( newVirtualPoint ); // logs "13, 56"

If the object or class has all the required properties, TypeScript will say they match, regardless of the implementation details.
